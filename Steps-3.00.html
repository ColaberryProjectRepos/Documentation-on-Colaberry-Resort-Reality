Data Warehouse Tables Description 3

 

3.  Data Model






  

3.1  Dim_Reservation



  

This section describes the data warehouse design, features and description of the fact and dimension tables.

3.1.1       Overview

The Reservation dimension contains the list of reservations, reservation type, canceled reservations, folio fees and other related attributes. Upgraded and other related attributes. The data is loaded from 5 Tables out of the 25 into the Staging Environment and then loaded into the Reservation Dimension table

3.1.2    Dependencies

The tables must be refreshed in the staging database first. The downstream dependency is for the two fact tables; Fact_Folio and Fact_SurveyScore.

3.2  Dim_Owner

   
  

3.2.1       Overview

The owner dimension contains the list of owners, owner type, owner status, Upgraded and other related attributes.  The data is loaded form 3 Tables out of the 25 into the Staging Environment and then loaded into the Owner Dimension table


3.2.2    Dependencies

The tables must be refreshed in the staging database first. The downstream dependency is for the three fact tables; Fact_Folio, Fact_Contract and Fact_ServiceCharge.

 

3.3  Dim_SiteLocation



  

3.3.1       Overview

The SiteLocation dimension contains the list of sites, site type, location, area, region and other related attributes. The data is loaded directly from t_Site table into Dim_SiteLocation. 


3.3.2    Dependencies

The table must be refreshed in the staging database first. The downstream dependency is for the two fact tables; Fact_Folio and Fact_Inventory.

 

3.4  Dim_Survey



 


3.4.1       Overview

The Survey dimension contains the list of Survey questions, answers, description, question type and other related attributes. Upgraded and other related attributes. The data is loaded from 10 Tables out of the 25 into the Staging Environment and then loaded into the Survey Dimension table.

3.4.2    Dependencies

The tables must be refreshed in the staging database first. The downstream dependency is for the one fact tables, Fact_SurveyScore. 

3.5  Fact_Folio


  
  

3.5.1     Overview

The Folio fact table contains the list of Folio Line item Amount, Foilo description, Transaction Code, Folio Line Item GL Account, Fin Trans Code, ACH Type and other related attributes. The data is loaded from 3 tables into the Staging Environment and then into the Fact Table. To avoid duplicate values while loading in the fact table using the Merge Statements, a merged key column has been used in this table which is generated as a combination of the Folio Line Item ID and Folio ID.  The fact table also includes foreign keys from the Dim_reservation, Dim_Owner and Dim_SiteLocation table.


3.5.2    Dependencies

The tables must be refreshed in the staging database first. The upstream dependency is for the three dim tables; Dim_reservation, Dim_Owner and Dim_SiteLocation.

 

3.6  Fact_Contract



  

3.6.1     Overview

The Contract fact table contains the list of Owner Number, Contract number, Contract Maintenance Fee, Contract Point Balance, Upgraded and other related attributes. The data is loaded from 3 tables into the Staging Environment and then into the Fact Table. The fact table includes foreign keys from the Dim_Owner table.


3.6.2    Dependencies

The tables must be refreshed in the staging database first. The upstream dependency is for only one dim table, Dim_Owner.

3.7  Fact_ServiceCharge


 
  

3.7.1     Overview

The Service Charge fact table contains the list of Owner Number, Amount, Revenue Type, Aging Bucket, Delinquent, Nbr Of Days Delinquent and other related attributes.  The data is loaded from 4 tables into the Staging Environment and then into the Fact Table. The fact table includes foreign keys from the Dim_Owner table.


3.7.2    Dependencies

The tables must be refreshed in the staging database first. The upstream dependency is for only one dim table, Dim_Owner.

 

3.8  Fact_ServiceScore





   

3.8.1     Overview

The SurveyScore Fact table contains the list of answer, AnswerValue, AccumulativeScore and other related attributes.  The data is loaded from 10 tables into the Staging Environment and then into the Fact Table. The fact table includes foreign keys from the Dim_Reservation and Dim_Survey table.


3.8.2    Dependencies

The tables must be refreshed in the staging database first. The upstream dependency is for two dim tables; Dim_Reservation and Dim_Survey.


3.8  Fact_Inventory



  

3.9.1     Overview

The Inventory Fact table contains the list of WKS, Bookings, Burned, Load To Book Days, Week Available and other related attributes. The data is loaded from 3 tables into the Staging Environment and then into the Fact Table. The fact table includes foreign keys from the Dim_SiteLocation table.


3.9.2    Dependencies

The tables must be refreshed in the staging database first. The upstream dependency is for only one dim table, Dim_SiteLocation.
