Designing a Data Warehouse 2

This section outlines the design of the CRR  Data Warehouse.

 2.1              Staging Tables

Not all the tables from the OLTP Database has been used in the data warehouse. The necessary tables to populate the data warehouse has been moved over to a staging database using a SQL Server Integration Services package. This package is Truncating all the records from Destination staging tables and then doing a full data load from OLTP daily, using SQL Server Agent.

2.2              Loading the Data Warehouse

This data warehouse is Designed using Galaxy Schema - which is an extension of a Star Schema. The star schema separates business process data into facts, which hold the measurable, quantitative data about a business, and dimensions which are descriptive attributes related to fact data. To Build a Data Model Using Galaxy Schema the steps are:


a. Determine the Facts and Dimensions

b. Denormalize the data

c. Create Dimension Tables

d. Create Fact Table
